当你部署了一个新的版本，那么当用户导航到一个新的页面时，用户的客户端会自动过渡到新部署的版本

> **它是如何工作的？** 每当检索静态资产返回一个 `404` 时，那么 vit-plugin-ssr 就会回落到 <Link text="Server Routing" href="/server-routing" />。当用户浏览到一个新的页面时，这将有效地重新加载整个客户端。(这样做是因为所有静态资产的 URL 都包含一个唯一的哈希值）。
>
> 它自动工作，你不需要做什么。
>
> 到目前为止，这种技术显示是可靠的，但如果它不适合你，那就开一个新的 GitHub issue。
