## vite-plugin-ssr{#vite-plugin-ssr}

使用 `vite-plugin-ssr` 的调试日志：

```shell
# 检查路由
DEBUG=vps:routing npm run dev
#检查找到了哪些 `.page.js` 文件及其处理方式
DEBUG=vps:pageFiles npm run dev
# 检查 HTTP Streaming
DEBUG=vps:stream npm run dev
```

> 这些调试日志让你了解 `vite-plugin-ssr` 如何处理应用程序。例如，要找出 404 的原因，可以使用 `vps:routing` 和 `vps:pageFiles`

还有 `vps:glob` / `vps:extractAssets` / `vps:extractExportNames`，不过这些是为 `vite-plugin-ssr` 的维护者准备的

> 如果你想要更多的调试信息，请在 [#249](https://github.com/brillout/vite-plugin-ssr/issues/249) 添加评论

## Vite{#vite}

把环境变量 `DEBUG` 设置为 `vite:*` 来开启 Vite 的调试日志

```shell
$ DEBUG=vite:* npm run dev
```

如果你发现了 Vite 的 bug，我们建议你直接在 `node_modules/vite/` 中检查 Vite 的源码，例如通过注入 `console.log()` ，使用二分法排查 bug

> 深入研究 Vite 的源码听起来可能很吓人，但实际上它是一种更快（也更有趣）的找到问题根源的方法。Vite 的源码优雅易读。它让你更接近问题所在。如果你开了 Vite 的 PR，随时联系 `vite-plugin-ssr` 的维护者，他们会让你的 PR 更快得到合并
